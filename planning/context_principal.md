Objetivo e contexto
Álvaro é o desenvolvedor especialista técnico por trás da BaseMedical, é um SaaS que tem como visão ser a mais abrangente plataforma para o setor de saúde, mas por enquanto foca no Agendamento de Profissionais da Saúde para seus Pacientes. Projetada para competir com a Doctoralia no mercado brasileiro. Sua esposa atua como Product Owner, fornecendo conhecimento especializado em saúde e definindo os requisitos de experiência dos Profissionais da Saúde e dos Pacientes. A plataforma permite que Profissionais de Saúde gerenciem agendamentos, criem perfis públicos, que servem como pequenas páginas web para um posicionamento na web, caso não o tenham, e se conectem com pacientes por meio de um sistema de agendamento sofisticado. Desenvolvida com Next.js 14, TypeScript, Prisma ORM com MySQL, NextAuth, Stripe para pagamentos e Twilio para notificações via WhatsApp, tudo implantado na Vercel com uma identidade visual consistente em tons de esmeralda e turquesa.

A plataforma suporta múltiplos tipos de perfil profissional (TOTAL para agendamento completo, INFO para perfis somente de contato, LISTA DE ESPERA para captura de leads) e planos de assinatura (GRATUITO, PROFISSIONAL, CORTESIA) com diferentes limites de recursos. O sucesso é medido pela criação de um sistema estável e pronto para produção que proporcione uma excelente experiência do usuário tanto para profissionais de saúde quanto para pacientes, com atenção especial aos padrões de design profissional e layouts responsivos.

Estado atual
Álvaro está resolvendo ativamente desafios técnicos em vários componentes do sistema. O trabalho recente inclui a transição da geração de PDFs de vouchers de cortesia do Puppeteer no servidor para o jsPDF no cliente, a fim de eliminar falhas de compilação e melhorar o desempenho. Ele está implementando o tratamento adequado de erros para entradas duplicadas de CPF no cadastro de usuários e redesenhando os cartões de perfil profissional com melhor formatação de endereço e exibição destacada dos números de registro profissional.
A plataforma passou por uma migração significativa de banco de dados do PostgreSQL para o MySQL para otimização de custos, o que exigiu extensas atualizações de esquema e correções de compilação do TypeScript. O desenvolvimento atual se concentra em manter a estabilidade do sistema, adicionando recursos sofisticados, como o sistema de vouchers de cortesia com geração em lote administrativa e fluxos de trabalho de ativação de usuários.

No horizonte
Álvaro planeja implementar um sistema abrangente de avaliação e classificação, onde os pacientes podem fornecer feedback sobre os profissionais de saúde, incluindo moderação de comentários, respostas profissionais e rastreamento da interação do usuário. O sistema suportará fluxos de trabalho de status pendente/aprovado/rejeitado para avaliações.

Outros recursos planejados incluem a expansão do sistema de tickets de suporte ("Chamados") para o relato centralizado de problemas e o aprimoramento contínuo da funcionalidade de busca e das páginas públicas para melhorar a posição competitiva em relação às plataformas de saúde estabelecidas.
Principais aprendizados e princípios
As decisões de arquitetura de banco de dados são orientadas por considerações práticas de custo - a migração do PostgreSQL para o MySQL demonstrou a importância de equilibrar recursos com despesas operacionais. Soluções do lado do cliente geralmente oferecem melhor desempenho e menos complicações de implantação do que alternativas do lado do servidor, como evidenciado pela transição na geração de PDFs.

Detalhes da experiência do usuário são muito importantes para plataformas profissionais - os números de registro profissional devem ser exibidos com destaque, os URLs devem usar nomes profissionais em vez de IDs de banco de dados para melhor UX e segurança, e a consistência visual em toda a identidade da marca esmeralda/turquesa é essencial. O tratamento de erros deve fornecer mensagens amigáveis ​​ao usuário em vez de expor restrições técnicas do banco de dados.

Abordagem e padrões
O desenvolvimento segue uma abordagem sistemática de identificação de erros de compilação, implementação de correções e testes com `npm run build` após cada alteração. As alterações no esquema do banco de dados são cuidadosamente planejadas para manter a integridade dos dados, ao mesmo tempo que oferecem suporte a novos recursos. A base de código usa padrões estabelecidos com componentes shadcn/ui, layouts de grade responsivos e esquemas de cores consistentes.

A resolução de problemas envolve estreita colaboração entre a implementação técnica e os requisitos do produto, com o Product Owner fornecendo conhecimento especializado na área da saúde, enquanto Álvaro cuida da arquitetura técnica. Funcionalidades críticas, como sistemas de agendamento, recebem atenção cuidadosa para evitar interrupções em recursos essenciais.
